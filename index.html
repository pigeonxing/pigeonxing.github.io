Hello world

发表于 2022-06-27 分类于 博客搭建 Valine：

Hello world!

这其实并不是这个博客第一篇 Hello world 的文章，之前各种测试文章删了又删，终于有了现在这个文章。

那么，随便谈谈我搭建这个博客究竟经历了什么吧：

（如果你不感兴趣，可以直接跳过下面部分内容）

初见 Hexo

最开始是下载了 node.js 和 git，之后安装了 Hexo。之后配置了 github 仓库，配置 SSH。

接着初始化 Hexo，再部署到 giyhub pages。这几步都还是挺简单的，并且博客确实可以用了。

但是呢？只是“能访问”吧。还是不太好用的。

疯狂踩坑与最终成型

最初我使用的主题是 matery，一个花里胡哨的主题。一步一步配置了各种界面，但是由于对于这个东西还不太熟悉，改配置文件改错了某处之后又改不回来的事故经常发生。这个时候只能是删除博客所在的文件夹，整个从头再来。那么我在配置什么呢？头像，博客头图。一个看起来很简单的东西，整的我几乎吐血。

无数次的 hexo clean + hexo generate + hexo server 三连之后，本地预览是没有问题了。但是，在 hexo deploy 命令发出后，我的博客却没能成功部署到 github pages。当时甚至没想到去 github pages 看看部署记录，只是觉得“欸，这个部署是不是有延迟啊？”。最后发现，我几个小时的数次部署都没有成功，不知道是网络原因还是配置原因。

最终，我花掉了几个小时的努力还是又一次删掉了。这一次我换了 NexT 主题，一个简约的主题，也很受欢迎，也很符合我的审美。NexT 的配置相对简单一点，因为功能简洁。配置了相关页面之后尝试部署了一次，其实它成功了。但是部署确实有延迟。而在等延迟的时间里，我又一次以为部署不成功，删除了整个文件夹。不过还好我突然醒悟，去 github pages 看了一眼部署记录，看到那个 Active 2 minutes ago 的时候，差点背过气。

再一次配置的时候就得心应手多了。配置好了各种页面、Valine 评论系统。但是由于查阅了过期的文档，填了古老的过期参数，导致博客的“中文”变成了不知道哪国的鬼画符语言。虽然文档过期了，但是它是官方文档啊！当时我没有意识到它过期了，还觉得“官方的文档肯定是对的啦，肯定是我配置错了啦”，于是删掉再来一次。又一次发现没啥子改变，才发现自己查阅了过期的文档。

之后的工作基本是稳步推进，查阅了无数资料之后学会了换博客背景图，并且把博客文章、菜单背景改透明。但是不知道哪里出了问题，博客的交互出了问题。很奇怪。如果为了美化就放弃博客的基本功能是不行的，所以最后我还是删掉了更改，手动把版本回滚到之前的那一版。

又查阅了很多资料之后，找到了最新版 Hexo 更改背景的正确方法。但是由于我坚持把资源放在本地，而 Hexo server 是通过本地服务器预览效果，所以最开始我没发现这个背景图加载很慢。但是部署后我的笔记本电脑、iPad、手机都不能快速加载背景图，属于加载很久后加载失败的那种。但是我的好兄弟试了一下，发现很快，甚至加载背景图比加载测试博客还快，秒加载。

很奇怪。由于这个不稳定性，我开始考虑我博客的图片怎么解决。最开始是使用了和背景图相同的本地存储方案，不过由于背景图很慢，所以我也不敢保证本地图片加载速度。毕竟我测试用的图片只有 40k，而背景图是 4M 的大小，所以当初测试图没问题不能保证之后没问题。

于是考虑是哪里可以优化。最开始我选择为我的博客加入 cdn 加速服务。在花了很久配置好之后，我发现访问速度极度不理想。后来才发现，我选择的服务商在一个月前 cdn 被污染了。尝试了解决方案之后发现还是没什么用。

后来我考虑使用 PicGo 加 github 搭建自己的图床。搭建好之后发现访问速度不理想，而且上传和加载不稳定。遂放弃自己乱搞，尝试使用成熟的图床。最开始选择 SM.MS 这个老牌图床，但是在经过我的测试后发现速度还是贼慢。后来选择了另一家图床，虽然不知道它能不能长期运营，会不会突然塌掉之类的，但是它加载速度非常快。虽然不能说秒加载，但是还是足够快。

现在，这个博客的基本功能差不多算是完成了。它有最基本的功能，可以渲染 markdown 文档；可以让我放图片来提供精彩的博客内容；支持评论系统；还有比较符合我审美的布局。最大的遗憾可能是我还没能加上 Mathjax，不支持 LaTeX 公式渲染；没能加上“一言”这个功能。

一点感想

虽然搞这个东西花了我很久，但是我还是坚持把它搞出来了。虽然心态爆炸也很常见，尤其是每一次部署的时候，很惊心动魄，很害怕出现一个 ERROR 或者 FATAL 之类的提示。不过这个也算是勉强让我找回了一点调试程序的快乐吧。虽然改博客和写程序不同，一个只需要改一改配置文件，而另一个需要大量思维，不过还是算有些共同点的吧。

因为我想要写下自己的感想，记录我想写的一切，虽然没什么意义，但是这算是我能提起精神做的少数几件事了吧。
